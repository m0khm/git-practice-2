# Постановка задачи

## Актуальность

Проект ProNet актуален, так как современные сервисы трудоустройства часто разделяют резюме, профессиональные связи, публикации, отклики и коммуникацию с работодателями между разными платформами. Из-за этого пользователю приходится использовать несколько отдельных сервисов, а работодателю сложнее получить целостное представление о кандидате.

ProNet объединяет функции профессиональной социальной сети и сервиса поиска работы. Пользователь может вести карьерный профиль, просматривать ленту, искать вакансии, отправлять отклики, получать уведомления и отслеживать статусы карьерных действий в одной системе.

## Ограничения

1. Проект выполняется на уровне проектирования и прототипирования.
2. Полная промышленная реализация системы не входит в рамки курсовой работы.
3. Основное внимание уделяется требованиям, пользовательским сценариям, диаграммам, модели данных и интерфейсу.
4. Интеграции с внешними сервисами, платежами и реальными базами работодателей рассматриваются как перспективы развития.

## Критерии успешности

1. Описаны проблема, цель, объект, предмет и задачи проекта.
2. Подготовлены функциональные и нефункциональные требования.
3. Разработаны пользовательские сценарии и проектные диаграммы.
4. Подготовлен прототип интерфейса ProNet.
5. Репозиторий оформлен и содержит основные проектные материалы.

## Риски

- Недостаточная детализация требований — снижается через разделение требований на функциональные, нефункциональные и бизнес-правила.
- Перегрузка интерфейса — снижается через выделение ключевых пользовательских сценариев.
- Размытость границ проекта — снижается через явное описание ограничений курсовой работы.
